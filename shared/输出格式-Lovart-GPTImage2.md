# 输出格式 · Lovart（底层 GPT-Image 2）

> 本文件是《4_页面输出.md》控制器选择 Lovart 引擎时调用的**输出格式定义**。
> 它只规定"每页提示词长什么样、版式定义怎么转译、配色与负面约束怎么写"。
> "读什么、怎么理解上游短稿、信息密度来源、咨询基调"等内容逻辑见《4_页面输出.md》第 5 节与 `00_全局约定.md`，本文件不重复定义，只引用。
> 品牌视觉值、对象坐标、尺寸、文字大小、字重、对齐方式以 `页面版式定义-IBM.md` 为唯一权威源；本文件不写入任何具体数值。

---

## 0. 引擎特性与总原则

Lovart 基于 GPT-Image 2，对**结构化、分层、显式约束**的提示词响应最稳定。因此本引擎采用「五层结构」逐页输出：① 基础层 / ② 风格层（含配色铁律）/ ③ 内容层 / ④ 排版参考 / ⑤ 负面约束。每页都必须五层齐全、可独立复制给 Lovart 使用。

- 配色铁律、负面约束必须**逐页重复**，不得只在文件开头统一声明（GPT-Image 对全局约束的记忆不稳定，逐页重复是对的）。
- 坐标、尺寸、pt 字号**只作内部对照**，必须按第 5 节转译为空间语言与文字层级语言后才进入提示词，原始数值不进入画面、不进入给 Lovart 的正文。
- `[ ]` 内为对你的生成指令，**不得**作为说明句出现在最终输出。

### P0-4 · 中文 / 专有名词渲染防护（GPT-Image 已知硬伤，最高优先级）

GPT-Image 在渲染**中文长句、专有名词、数字**时存在已知硬伤：长句易掉字 / 错字 / 串行；SAP 模块名、客户全称、关键数字易被"美化改写"成形近词或随手编的英文。为对抗这一点：

- **③ 内容层**必须在每页提示词中**单列一份「精确渲染清单（verbatim list）」**：把本页所有关键专有名词（SAP 模块名 / 产品名、客户全称、关键数字与量级、来源标注）逐条列出，明确要求 Lovart **逐字 1:1 渲染、不得改写、不得拼错、不得翻译、不得增减字符**。该清单是渲染校验锚点，画面文字必须与清单逐字一致。
- **⑤ 负面约束**必须补一条：防止专有名词 / SAP 模块名 / 客户全称 / 数字被改写、拼错、形近替换或自行翻译。
- 此防护对四类页面均生效（封面 / 目录 / 章节页 / 正文页），只要页面含专有名词或数字即须出「精确渲染清单」。

> 信息密度、咨询基调、维度词/枚举词禁令等**推理级**约束统一引用 `00_全局约定.md`，本文件不复述（见各层标注）。信息密度由上游 Solution 阶段固化，Pages 阶段只读取、不改档。

## 1. 输出文件首行

每个输出文件第一行必须固定为：

```markdown
根据我给你的页面描述，并行生成 PPT 图片。
```

该句只在整个输出文件首行出现一次；后续每个 Slide 区块不得重复输出该句。首行之后，每个页面先按《4_页面输出.md》第 3 节映射判断页面类型，再按下方对应的五层结构格式输出。

判断规则：

- `【PXXX · 封面】` → L1 封面格式。
- `【PXXX · 目录】` → L2 目录格式。
- `【PXXX · 章节页】` → L3 章节页格式。
- `【PXXX】页面标题（Action Title）` → L4 正文页格式。

> 以下四类模板共用同一「五层结构」。坐标、对象尺寸、文字大小、字重、对齐方式与品牌值从 `页面版式定义-IBM.md` 读取引用（见第 4、5 节），不得在本文件内直接写入或复制具体数值。

## 5. 版式定义引用与转译约定

读取版式定义中的对象位置、尺寸、文字大小、字重与对齐方式后，**不得**把原始 X/Y/W/H、尺寸数字或 pt 字号写进给 Lovart 的提示词；必须按对象名引用权威定义，并转成空间语言与文字层级语言，填入 ④ 排版参考：
